Abstract

AISLA (AI Self-Learning Lab Assistant) is an innovative full-stack educational platform designed to revolutionize laboratory learning experiences for students and faculty. The platform addresses the critical gap between theoretical knowledge and practical application by providing AI-powered experiment generation, interactive 3D chemistry simulations, and real-time learning assistance.

The system solves key challenges in traditional lab education including limited access to physical laboratories, passive learning experiences, lack of instant support, and manual content creation overhead. Using cutting-edge technologies like React.js, Node.js, MongoDB, Babylon.js for 3D rendering, and integration with Ollama/Google Gemini for AI capabilities, AISLA delivers a comprehensive solution that enables students to learn anytime, anywhere with intelligent guidance and safe virtual experimentation.

Problem Statement

What is the Issue?

Traditional laboratory education faces significant barriers that prevent effective learning. Students often struggle with limited lab access, safety concerns, and lack of real-time support when studying complex experiments.

Why is it a Problem?

  • Limited Lab Access: Physical labs have fixed hours, limited equipment, and capacity constraints. Students cannot practice outside scheduled sessions.
  • Passive Learning: Reading lab manuals and watching demonstrations does not engage students effectively or help them retain procedural knowledge.
  • No Instant Support: When students have doubts while studying experiments, they must wait for instructors, breaking their learning flow.
  • Safety Concerns: Chemistry experiments involve hazardous materials. Fear of accidents prevents free experimentation.
  • Manual Content Creation: Faculty spend excessive time creating structured experiment documentation, quizzes, and assessments manually.
  • Viva Anxiety: Students feel unprepared for practical viva examinations due to lack of practice and feedback.

Who is Facing the Problem?

  • Students: Engineering, science, and medical students who need to understand laboratory procedures and concepts
  • Faculty: Teachers and professors who spend hours creating experiment content and assessments
  • Institutions: Universities and colleges with limited lab infrastructure or resources

Data Flow

  1. User Request: Browser sends HTTP request with JWT token to backend API
  2. Authentication: Middleware verifies JWT and populates user context
  3. Business Logic: Controller processes request, calls services as needed
  4. AI Processing: AI Service connects to Ollama/Gemini for content generation
  5. Database: Mongoose models handle MongoDB CRUD operations
  6. Response: JSON response sent back to frontend for rendering

Implementation Details

AI Service Implementation

The AI service (aiService.js) handles all AI-related functionality:

  • Multi-Provider Support: Flexible configuration allows switching between Ollama (local) and Gemini (cloud)
  • Structured Prompts: Carefully crafted prompts ensure consistent JSON output
  • Streaming: SSE-based streaming for real-time token display
  • Fallback Handling: Default content returned if AI fails

3D Chemistry Lab Implementation

Built with Babylon.js (ChemistryLab.js - 1006 lines):

  • Scene Setup: Initialize 3D scene with lighting, shadows, camera
  • Equipment Creation: Programmatically create beakers, test tubes, burner using primitives
  • Particle Systems: Visual effects for reactions (bubbles, smoke)
  • Reaction Engine: Backend service matches chemicals ‚Üí returns reaction data

Real-Time Chat Implementation

  • Server-Sent Events: Backend connects to Ollama with stream:true
  • Token Relay: Each token sent immediately to client
  • Markdown Parsing: Custom parser for rich text rendering

Code Execution Implementation

  • Piston API: External service for secure sandbox execution
  • Language Config: Template code and version info per language
  • Test Cases: Run code against input ‚Üí compare output

Faculty-Student Chat Implementation

Built with Socket.IO for real-time bidirectional communication:

  • Socket.IO Server: WebSocket server integrated with Express for real-time messaging
  • JWT Authentication: Tokens verified on WebSocket handshake for secure connections
  • Room Management: Dynamic rooms for each faculty-student chat conversation
  • Typing Indicators: Real-time broadcast of typing status to chat participants
  • Read Receipts: Message delivery and read status tracking
  • Doubt/Task Tracking: Messages tagged as doubts or tasks with resolution/status management

Virtual Physics Lab Implementation

  • 3D Simulation Engine: Interactive physics simulations for mechanics experiments
  • Optics Module: Light refraction, reflection, and lens experiments
  • Electromagnetism: Circuit simulations and electromagnetic field visualization
  • Measurement Tools: Virtual instruments for taking readings

AR Chemistry Camera Implementation

  • Device Orientation API: Gyroscope support for pseudo-AR experience
  • Camera Integration: MediaDevices API for camera access
  • Chemical Overlay: 3D molecules rendered over camera feed
  • Reaction Visualization: Real-time reaction effects on mobile devices

Conclusion

AISLA (AI Self-Learning Lab Assistant) successfully demonstrates how artificial intelligence can transform traditional laboratory education. By combining AI-powered content generation, interactive 3D simulations, and real-time assistance, the platform addresses critical challenges in lab learning including limited access, safety concerns, and lack of instant support.

The project achieved all its primary objectives: automated experiment content creation (reducing hours to minutes), safe virtual chemistry experiments, real-time AI chat support, automated quiz generation, and an integrated coding environment. The successful deployment at https://labs-m2g6.onrender.com/ proves the viability of the solution.

Key Learnings

  • Integration of multiple AI providers (Ollama/Gemini) provides flexibility and cost optimization
  • Server-Sent Events enable smooth real-time streaming experiences
  • Babylon.js provides powerful 3D capabilities for web-based simulations
  • Role-based access control is essential for multi-user educational platforms
  • MongoDB's flexible schema suits AI-generated dynamic content well

AISLA represents a significant step toward democratizing laboratory education, making quality learning accessible to students regardless of their location or institution's resources. The platform is ready for real-world deployment and can serve as a foundation for more comprehensive virtual learning environments.
