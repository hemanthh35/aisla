Project Overview

üìñ What is AISLA?

AISLA (AI Self-Learning Lab Assistant) is a next-generation, full-stack educational platform that fundamentally transforms how students learn and practice laboratory experiments. Built specifically for the GITAM Hackathon, this innovative platform combines the power of artificial intelligence, 3D visualization, and interactive simulations to create an immersive learning environment that bridges the gap between theoretical knowledge and practical application.

Unlike traditional learning management systems, AISLA doesn't just deliver content‚Äîit understands it. Using advanced Large Language Models (LLMs), the platform can analyze lab manuals, generate structured experiment content, create intelligent quizzes, answer student queries in real-time, and even simulate chemical reactions in a virtual 3D laboratory environment.

üî¥ The Problem We Solve

Traditional laboratory education faces several critical challenges that hinder effective learning:

  • Limited Lab Access: Students often have restricted access to physical laboratories due to time constraints, resource limitations, or geographical barriers. Many institutions lack adequate lab facilities.
  • Passive Learning: Reading lab manuals and watching demonstrations is a passive experience that doesn't engage students effectively or help them retain complex procedures.
  • No Instant Support: When students have doubts while studying experiments, they must wait for the next class or office hours to get answers, breaking their learning flow.
  • Manual Content Creation: Faculty spend countless hours creating structured experiment documentation, quizzes, and assessment materials manually.
  • Safety Concerns: Chemistry and physics experiments can be dangerous. Students may hesitate to experiment freely due to safety concerns.
  • Lack of Practice: Without access to labs, students cannot practice coding, conduct experiments, or visualize scientific concepts effectively.
  • Viva Anxiety: Students often feel unprepared for practical viva examinations due to lack of practice and personalized feedback.

AISLA addresses each of these challenges with a comprehensive suite of AI-powered tools:

Key Features (Detailed)

AISLA is packed with powerful features designed to transform the laboratory learning experience. Each feature is carefully crafted to address specific educational challenges and provide maximum value to students, faculty, and administrators.

üß™ 1. AI-Powered Experiment Generation

What It Does

Automatically generates comprehensive, structured experiment documentation from minimal input. Faculty simply provide a title and brief description‚Äîthe AI creates everything else.

Generated Content Includes:

  • Aim: Clear objective statement describing what the experiment seeks to achieve
  • Theory: Comprehensive theoretical background with relevant scientific principles
  • Apparatus List: Complete list of required equipment and materials
  • Procedure: Step-by-step instructions with numbered, actionable steps
  • Key Formulas: All mathematical equations needed for calculations
  • Observations: Expected observations and data recording format
  • Precautions: Safety guidelines and best practices
  • Common Mistakes: What to avoid and troubleshooting tips
  • Real-World Applications: Practical uses of the experiment's concepts
  • Summary: Quick recap for revision purposes

How It Works:

  1. Faculty enters experiment title (e.g., "Ohm's Law Verification")
  2. Optionally adds brief content or uploads a lab manual image
  3. AI (Ollama or Gemini) processes the input with structured prompts
  4. System generates JSON-structured experiment with all sections
  5. Content is saved to MongoDB with faculty's ID as creator
  6. Real-time progress shown via Server-Sent Events (SSE)

üî¨ 2. Virtual 3D Chemistry Laboratory

What It Does

A fully immersive 3D virtual chemistry laboratory built with Babylon.js where students can conduct experiments safely. Mix chemicals, heat solutions, observe reactions, and learn through interactive experimentation‚Äîwith zero physical risk.

Lab Equipment Available:

  • Beakers: Various sizes for mixing and containing solutions
  • Test Tubes: Multiple test tubes in racks for parallel experiments
  • Conical Flasks: For titration and mixing experiments
  • Measuring Cylinders: Precise volume measurement equipment
  • Bunsen Burner: Toggleable heat source with realistic flame animation
  • Tripod Stand: For heating experiments over the burner
  • Stirring Rod: For mixing solutions

Reaction Simulation Features:

  • 50+ Chemical Reactions: Pre-configured with accurate visual effects
  • Visual Effects: Bubbles, smoke, precipitates, color changes, gas emission
  • Heat-Dependent Reactions: Some reactions require the Bunsen burner
  • Safety Warnings: Dangerous combinations trigger safety alerts
  • Particle Systems: Realistic particle effects for reactions
  • Chemical Equations: Balanced equations displayed during reactions

How It Works:

  1. Student selects chemicals from the available inventory panel
  2. Clicks on containers (beakers/test tubes) to add chemicals
  3. System checks for reactions via the ReactionEngine
  4. If reaction exists, visual effects are triggered (bubbles, color change, etc.)
  5. Explanation and balanced equation are displayed
  6. Heat can be applied using the Bunsen burner for heat-dependent reactions

ü§ñ 4. Real-Time AI Chat Assistant

What It Does

A ChatGPT-like streaming chat widget that provides instant answers to student queries. The assistant understands context, provides detailed explanations, and helps students learn concepts in real-time‚Äîavailable 24/7 on every page.

Key Capabilities:

  • Streaming Responses: Real-time token-by-token display with typing effect
  • Markdown Support: Bold, italic, code blocks, lists, and formatting
  • Conversation History: Maintains context across multiple messages

Technical Implementation:

  • Uses Server-Sent Events (SSE) for real-time streaming
  • Backend connects to Ollama's /api/chat endpoint with stream: true
  • Tokens are relayed to frontend as they're generated
  • Custom markdown parser for rich text rendering

Security & Authentication

üîê Authentication Flow

  1. User submits credentials ‚Üí POST /api/auth/register or /login
  2. Server hashes password with bcryptjs (10 salt rounds)
  3. Server generates JWT token with user._id
  4. Frontend stores token in localStorage
  5. All protected requests include: Authorization: Bearer <token>
  6. Middleware verifies token and populates req.user
